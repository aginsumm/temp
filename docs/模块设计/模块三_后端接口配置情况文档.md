## 七、部署说明

### 7.1 开发环境部署

**后端部署**：
```bash
cd backend
pip install -r requirements.txt
python init_data.py  # 初始化示例数据
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**前端部署**：
```bash
cd frontend
npm install
npm run dev
```

**访问地址**：
- 后端API：http://localhost:8000
- 前端界面：http://localhost:5173
- API文档：http://localhost:8000/docs

### 7.2 生产环境部署

**后端部署**：
1. 修改 `.env` 文件配置生产数据库（PostgreSQL）
2. 使用Gunicorn或uWSGI部署FastAPI应用
3. 配置Nginx反向代理
4. 启用HTTPS

**前端部署**：
1. 运行 `npm run build` 构建生产版本
2. 将构建产物部署到静态文件服务器
3. 配置Nginx提供静态文件服务

## 九、已知问题和限制

### 9.1 功能限制

1. **数据规模限制**
   - 当前实现使用SQLite，适合中小规模数据
   - 大规模数据建议使用PostgreSQL

2. **搜索功能限制**
   - 当前仅支持简单的关键词搜索
   - 不支持模糊匹配和语义搜索

3. **图谱渲染限制**
   - 大规模图谱（>1000节点）可能存在性能问题
   - 建议使用虚拟滚动或WebGL渲染

### 9.2 优化建议

1. **数据库优化**
   - 为常用查询字段创建索引（type, region, period）
   - 使用分页查询避免一次性加载过多数据

2. **缓存策略**
   - 使用Redis缓存频繁访问的图谱数据
   - 缓存搜索结果，设置合理的过期时间

3. **前端优化**
   - 大规模图谱使用虚拟滚动
   - 使用WebGL渲染提升性能
   - 实现代码分割和按需加载
