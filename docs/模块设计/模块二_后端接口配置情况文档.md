## 四、数据模型

### 4.1 会话模型（Session）

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| id | String | 会话唯一标识 | 是（自动生成） |
| user_id | String | 用户ID | 是 |
| title | String | 会话标题 | 是（默认"新对话"） |
| created_at | DateTime | 创建时间 | 是（自动生成） |
| updated_at | DateTime | 更新时间 | 是（自动生成） |
| message_count | Integer | 消息数量 | 是（默认0） |
| is_pinned | Boolean | 是否置顶 | 是（默认False） |

### 4.2 消息模型（Message）

| 字段 | 类型 | 说明 | 必填 |
|------|------|------|------|
| id | String | 消息唯一标识 | 是（自动生成） |
| session_id | String | 会话ID | 是 |
| role | Enum | 角色（user/assistant） | 是 |
| content | Text | 消息内容 | 是 |
| created_at | DateTime | 创建时间 | 是（自动生成） |
| feedback | String | 反馈 | 否 |
| is_favorite | Boolean | 是否收藏 | 是（默认False） |

### 4.3 辅助模型

- **MessageSource**：消息来源（知识溯源）
- **MessageEntity**：消息实体（实体识别）
- **MessageKeyword**：消息关键词
- **FavoriteQuestion**：收藏问题

## 七、部署说明

### 7.1 开发环境部署

**后端部署**：
```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**前端部署**：
```bash
cd frontend
npm install
npm run dev
```

**访问地址**：
- 后端API：http://localhost:8000
- 前端界面：http://localhost:5173
- API文档：http://localhost:8000/docs

### 7.2 生产环境部署

**后端部署**：
1. 修改 `.env` 文件配置生产数据库（PostgreSQL）
2. 配置 `DASHSCOPE_API_KEY` 启用真实AI响应
3. 使用Gunicorn或uWSGI部署FastAPI应用
4. 配置Nginx反向代理
5. 启用HTTPS

**前端部署**：
1. 运行 `npm run build` 构建生产版本
2. 将构建产物部署到静态文件服务器
3. 配置Nginx提供静态文件服务

## 九、已知问题和限制

### 9.1 功能限制

1. **用户认证缺失**
   - 当前使用默认用户ID（default_user）
   - 不支持多用户数据隔离

2. **AI响应依赖**
   - 需要配置DASHSCOPE_API_KEY才能使用真实AI响应
   - 未配置时使用mock响应

3. **NLP功能待完善**
   - 实体识别、关键词提取使用静态mock数据
   - 需要集成真实NLP服务

### 9.2 优化建议

1. **数据库优化**
   - 为常用查询字段创建索引（user_id, session_id, created_at）
   - 使用分页查询避免一次性加载过多消息

2. **缓存策略**
   - 使用Redis缓存会话列表和消息
   - 缓存推荐问题，减少数据库查询

3. **性能优化**
   - 实现消息懒加载
   - 优化大文本消息的渲染性能
